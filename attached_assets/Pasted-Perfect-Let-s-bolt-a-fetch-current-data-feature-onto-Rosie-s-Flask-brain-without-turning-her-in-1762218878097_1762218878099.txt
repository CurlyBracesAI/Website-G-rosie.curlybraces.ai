Perfect. Let’s bolt a “fetch current data” feature onto Rosie’s Flask brain — without turning her into a rogue internet crawler.
Here’s the practical, production-safe way to do it:

🧱 Step 1 — Add a New Endpoint
We’ll create /rosie-online, which works like /rosie-test but adds a web-fetch step before the LLM call.
from flask import Flask, request, jsonify
import os, requests
from llm_router import call_llm

app = Flask(__name__)

# -------------------------
# 🔎 Controlled Web Fetcher
# -------------------------
def get_latest_data(query):
    """Fetch current info via SerpAPI or any chosen API."""
    serp_api_key = os.getenv("SERP_API_KEY")
    api_url = f"https://serpapi.com/search.json?q={query}&api_key={serp_api_key}&tbm=nws"
    try:
        r = requests.get(api_url, timeout=10)
        data = r.json()
        snippets = [n["snippet"] for n in data.get("news_results", [])[:3]]
        return "\n".join(snippets) if snippets else "No current data found."
    except Exception as e:
        return f"Error fetching data: {e}"

# -------------------------
# 🧠 Rosie-Online Endpoint
# -------------------------
@app.route("/rosie-online", methods=["POST"])
def rosie_online():
    data = request.get_json()
    user_message = data.get("input", "")
    provider = data.get("provider", "openai")

    # Step 1: Fetch latest data
    fetched_info = get_latest_data(user_message)

    # Step 2: Feed that data into Rosie's prompt
    combined_prompt = f"""
The user asked: {user_message}

Here is the latest relevant information I found:
{fetched_info}

Using this data and your existing knowledge, answer clearly and cite that this is current public info.
"""

    output = call_llm(
        user_message=combined_prompt,
        system_prompt="You are Rosie, an AI assistant with limited controlled internet access. Use fetched info responsibly.",
        provider=provider
    )

    return jsonify({
        "reply": output,
        "fetched_data": fetched_info
    })


🧩 Step 2 — Add the API Key
In Replit → Secrets, add:
Key: SERP_API_KEY
Value: your-real-serpapi-key

You can get one free from https://serpapi.com.

🪄 Step 3 — Client-side Control
In your chat UI JavaScript, add a simple checkbox:
<label><input type="checkbox" id="allowSearch"> Include live internet data</label>

Then adjust the fetch() call:
const allowSearch = document.getElementById('allowSearch').checked;
const endpoint = allowSearch ? "/rosie-online" : "/rosie-test";

So the user decides whether Rosie gets “live” data access.

🧠 Step 4 — Behavior


If the box is unchecked → uses standard offline GPT knowledge.


If checked → your Flask app fetches a short news summary and feeds it into Rosie.


Rosie’s tone and persona stay the same; she just gains context.



That’s it — you’ve now given Rosie supervised internet awareness.
